# 报告输出模板

Agent 生成报告时必须遵循此结构。

```markdown
# [视频标题]

> UP主：[name] | 时长：[duration] | BV：[bvid] | 链接：[url]

## 一句话总结

[30 字以内核心观点]

## 核心干货（5-10 条）

1. [要点]
2. [要点]
3. ...

## 详细要点

### [章节/话题 1]

- [细节]

### [章节/话题 2]

- [细节]

## 优质资源与链接

| 类型 | 名称 | 链接/出处 |
|------|------|-----------|
| 工具 | ... | ... |
| 文档 | ... | ... |
| 书籍/课程 | ... | 视频 [时间戳] |

## 高赞评论洞察

- **共识**：[观众普遍认同的观点]
- **补充**：[评论中有价值的补充信息]
- **争议**：[存在分歧的点]

## 思维导图

```mermaid
mindmap
  root(([视频主题]))
    [分支1]
      [子点]
    [分支2]
      [子点]
    实用资源
      [资源名]
```

## 附录：信息来源

- 字幕/转录：[subtitle / faster-whisper]
- 评论采样：Top N 条高赞评论
- B 站 AI 摘要：[已参考 / 未提供]
```

## 写作要求

- 干货条目要可行动，避免空泛复述标题
- 资源链接从逐字稿与评论中提取，标注大致时间戳
- 思维导图层级 2-3 层，覆盖主题、论点、资源三类
- 若仅有部分数据源（如无字幕且 ASR 失败），在附录中如实说明
